1 # Flexible, pipeline-based input handling.
2 
3 `iota` is a small, lightweight abstraction over the default WPILib functions
4 for handling joystick inputs, and extends its functionality to any kind
5 of input, whether through SmartDashboard, joystick, or sensor. Any value
6 that can be polled can be handled by `iota`.
7 
8 Furthermore, `iota` provides an elegant, simplistic pipeline implementation
9 in which reusable steps can be chained together. Often, inputs are not
10 directly applied to motor speeds; they are modified somehow. With `iota`,
11 this occurs by passing the input through a pipeline and then running a
12 callback with the resulting, transformed input.
13 
14 ### Notes on the name.
15 `iota` started out simply named `io`. However, this conflicts with `io`,
16 which is a Python core library, so trying to import `io` would implement the
17 core library instead of what is actually wanted. The word "iota" contains
18 "io", means "a very small amount" (the main code, `iota.py`, is only 72 SLOC
19 uncommented), and is pretty easy to type (being 4 letters long),
20 so I just decided to name it that.
21 
22 ### Notes on the language
23 `iota`, like all the other code written by FRC Team 5002 from the 2017-2018
24 season onwards, is written in Python. In fact, `iota` was a major reason
25 why we switched from Java to Python; Python allowed us much more flexibility
26 with its dynamic typing and tuples than Java ever could. In fact, just
27 the pipeline implementation in Java was already longer than the entire `iota`
28 in Python, and heavily generics-based and unreadable. With Python, "it just
29 works".
30 
31 ### Notes on the usage
32 Most detailed information on the usage of `iota` can be found in this
33 documentation. This is only meant as a quick overview.
34 
35 `iota` is used by constructing a pipeline and adding it to the poller.
36 For example,
37 
38 ```python
39 from iota import *
40 
41 pipe = Pipeline()
42 pipe.add(lambda x : not x, float)
43 Poller.on(gamepad1.x, Events.PRESSED, pipe, setServoSpeed)
44 ```
45 
46 where `gamepad1.x` is a function that returns the value of `x`,
47 Events.PRESSED is an enum value telling the poller to only run the pipeline
48 when a PRESSED action has been detected, `pipe` is the pipeline of steps,
49 and `setServoSpeed` is an example callback that actually applies the changes
50 to the hardware.
